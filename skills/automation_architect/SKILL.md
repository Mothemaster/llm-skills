# Automation Architect

## Purpose
Use this skill to design robust multi-step automation systems across tools and teams.

## Inputs
- Business process
- Systems involved
- Scale requirements
- Security or governance constraints

## Process
1. Map the current process.
2. Identify automation candidates and decision points.
3. Define system boundaries and handoffs.
4. Design fallback paths and observability.
5. Recommend architecture that scales.

## Output
- Architecture proposal
- Workflow map
- Tooling recommendations
- Risks, dependencies, and rollout plan

## Quality bar
- Favor clear systems over clever complexity.
- Include human review where needed.
- Make monitoring and recovery explicit.

# AI Agent Architect

## Purpose
Use this skill to design AI agents that can reason within scope, use tools safely, and collaborate in structured workflows.

## Inputs
- Agent goal
- Tools available
- Memory needs
- Failure tolerance
- Single-agent or multi-agent setup

## Process
1. Define the agent's responsibility boundaries.
2. Decide what the agent should know, remember, and ignore.
3. Specify tool invocation rules.
4. Add validation and fallback mechanisms.
5. Design orchestration only where it adds real value.

## Output
- Agent architecture
- Role definitions
- Tool-use policy
- Memory strategy
- Safety and reliability notes

## Quality bar
- Keep agents narrow and useful.
- Avoid fake autonomy.
- Add explicit checks around important actions.

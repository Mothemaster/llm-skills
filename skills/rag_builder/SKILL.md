# RAG Builder

## Purpose
Use this skill to design systems that retrieve the right knowledge and ground answers in source material.

## Inputs
- Knowledge sources
- Query types
- Latency or scale constraints
- Accuracy requirements

## Process
1. Define the retrieval use case.
2. Choose chunking and metadata strategy.
3. Plan embedding and indexing.
4. Improve recall and relevance.
5. Ground final responses in retrieved evidence.

## Output
- RAG architecture
- Chunking/indexing recommendations
- Retrieval and reranking notes
- Evaluation considerations

## Quality bar
- Optimize for relevance, not just ingestion volume.
- Keep provenance visible.
- Reduce hallucination risk with grounding.
